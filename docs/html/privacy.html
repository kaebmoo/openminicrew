1. About This Application

OpenMiniCrew is an open-source AI-powered personal assistant delivered as a Telegram bot. It provides tools for email summaries, calendar management, expense tracking, QR code generation, web search, and other everyday tasks.

This privacy policy describes what personal data OpenMiniCrew collects from its users, how that data is used, how long it is retained, and how users can control or delete their data. This policy applies to all users who interact with the OpenMiniCrew Telegram bot.

3. Google User Data — Detailed Disclosure

OpenMiniCrew accesses Google user data through the Gmail API and Google Calendar API. Access is granted only after you explicitly authorize it through the standard Google OAuth 2.0 consent flow.

3.1 What Google data we access

  • Gmail (read-only scope): We read email message headers (sender, subject, date) and body content solely to generate email summaries for you. OpenMiniCrew does not send, forward, modify, or delete any of your emails.
  • Google Calendar (events scope): We read your calendar events (title, date/time, description) to show your schedule. We create or delete calendar events only when you explicitly request it through the bot.

3.2 How we use Google data

  • Email content is sent to an AI language model (Google Gemini or Anthropic Claude, as configured) to generate a natural-language summary.
  • Calendar data is formatted and displayed to you, or sent to the AI model when you ask questions about your schedule.
  • After processing, email and calendar content is not stored permanently. Only minimal metadata (message IDs) is retained temporarily for deduplication.

3.3 Who we share Google data with

Your Google data is never sold, rented, or shared with any third party for advertising, marketing, or any purpose unrelated to providing the bot's features.

Google data is transmitted only to:

  • Google Gemini API or Anthropic Claude API (depending on your configuration) for the sole purpose of generating AI summaries of your emails or calendar events in response to your requests.

No other parties receive your Google data.

3.4 How to revoke access

You can revoke OpenMiniCrew's access to your Google account at any time by:

  • Sending /disconnectgmail to the bot (deletes the stored OAuth token)
  • Visiting your Google Account permissions page and removing OpenMiniCrew

Google API Services User Data Policy Compliance

OpenMiniCrew's use and transfer to any other app of information received from Google APIs will adhere to the Google API Services User Data Policy, including the Limited Use requirements.

Specifically, OpenMiniCrew:

  • Only uses Google user data to provide and improve the email summary and calendar features described in this policy.
  • Does not use Google user data for advertising or to serve ads.
  • Does not allow humans to read Google user data, except with the user's explicit consent, for security purposes, to comply with applicable law, or for the app's internal operations (limited to aggregated, anonymized data).
  • Does not transfer Google user data to third parties except as necessary to provide the features, with user consent, or for legal/security reasons.

4. Data Security

OpenMiniCrew implements the following security measures to protect your data:

  • Encryption at rest: Phone numbers, national IDs, expense notes, API keys, and Google OAuth tokens are encrypted using Fernet symmetric encryption before storage.
  • Webhook security: The Telegram webhook endpoint is protected by a secret token that is verified on every incoming request.
  • Log minimization: Operational logs store metadata (hashes, sizes, kinds) instead of raw message content.
  • Auto-deletion of sensitive messages: Messages containing API keys, phone numbers, or national IDs are automatically deleted from the Telegram chat after processing.
  • Admin data isolation: The system architecture prevents the bot administrator from accessing other users' encrypted private data.

5. Data Retention

Data is not kept indefinitely. The following retention controls are in place:

  • Chat history: Retained for a configurable number of days (default: 30 days), then automatically deleted.
  • Email metadata: Message IDs retained for deduplication are automatically cleaned up after a configurable period (default: 90 days).
  • Location: Expires automatically after a configurable time-to-live (default: 60 minutes).
  • Tool logs: Operational logs are automatically cleaned up after a configurable period (default: 90 days).
  • Full deletion: You can permanently delete all your data at any time using the /delete_my_data confirm command.

6. Your Rights and Controls

You have full control over your data through the following bot commands:

The /delete_my_data confirm command removes all user-linked records including: account data, chat history, conversations, expense records, to-do items, reminders, schedules, location data, API keys, email processing records, and Gmail OAuth tokens.
