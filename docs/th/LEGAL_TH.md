## ส่วนที่ 1: ข้อกำหนดการใช้งาน (Terms of Service)

### 1.1 คำนิยาม

- **"บริการ"** หมายถึง OpenMiniCrew Telegram Bot รวมถึง feature, เครื่องมือ (tools) และระบบที่เกี่ยวข้องทั้งหมด
- **"ผู้ให้บริการ"** หมายถึง บุคคลหรือทีมที่พัฒนาและดำเนินการ OpenMiniCrew
- **"ผู้ใช้"** หมายถึง บุคคลที่ใช้งานบริการผ่าน Telegram
- **"ข้อมูลส่วนบุคคล"** หมายถึง ข้อมูลที่สามารถระบุตัวบุคคลได้ทั้งทางตรงและทางอ้อม ตามนิยามใน พ.ร.บ. คุ้มครองข้อมูลส่วนบุคคล พ.ศ. 2562 (PDPA)
- **"AI/LLM"** หมายถึง โมเดลภาษาขนาดใหญ่ที่ใช้ประมวลผลข้อความ เช่น Claude (Anthropic), Gemini (Google)

### 1.2 ลักษณะของบริการ

OpenMiniCrew เป็นผู้ช่วยส่วนตัวผ่าน Telegram Bot ที่ใช้ AI ประมวลผลคำสั่งและข้อความของผู้ใช้ เพื่อเรียกเครื่องมือต่างๆ ได้แก่:

- สรุปอีเมล Gmail และอีเมลที่ทำงาน
- ปฏิทิน Google Calendar
- อัตราแลกเปลี่ยน (ธนาคารแห่งประเทศไทย)
- สร้าง PromptPay QR Code
- ตรวจผลสลากกินแบ่งรัฐบาล
- ค้นหาสถานที่ แผนที่ สภาพจราจร
- สภาพอากาศ
- ค้นหาข้อมูลจากเว็บ สรุปข่าว
- บันทึกรายรับรายจ่าย รายการสิ่งที่ต้องทำ การแจ้งเตือน
- แปลงหน่วย
- ตั้งเวลางานอัตโนมัติ (schedule)

### 2.6 ความยินยอม (Consent)

#### ระบบจัดการความยินยอม

บริการนี้ใช้ระบบ consent แบบ explicit สำหรับข้อมูล 3 ประเภท:

1. **ประวัติการสนทนา (chat_history)**
   - ผู้ใช้เดิม: ค่าเริ่มต้นคือ granted (เพื่อความต่อเนื่อง)
   - ผู้ใช้ใหม่: ค่าเริ่มต้นคือ not_set (ต้อง grant เอง)
   - ถ้า revoke: ลบประวัติแชททั้งหมดทันที + หยุดเก็บข้อมูลใหม่
   - Bot ยังทำงานได้แต่จะไม่มีบริบทสนทนาก่อนหน้า

2. **ตำแหน่ง GPS (location_access)**
   - ค่าเริ่มต้นคือ not_set (ต้อง grant ก่อนใช้)
   - ถ้า revoke: ลบตำแหน่งที่เก็บไว้ทันที
   - เครื่องมือที่ต้องใช้ตำแหน่ง (สภาพอากาศ, สถานที่) จะไม่ทำงานจนกว่าจะ grant

3. **Gmail/Calendar (gmail_access)**
   - ค่าเริ่มต้นคือ not_set
   - การ grant ต้องผ่าน OAuth flow จริง (/authgmail)
   - ถ้า revoke (/disconnectgmail): ลบ OAuth token + ยกเลิกการเชื่อมต่อทันที

#### วิธีจัดการ consent

- ดูสถานะ: `/consent`
- เปิด/ปิด: `/consent [chat|location|gmail] [on|off]`
- ดูภาพรวม: `/privacy`

## ส่วนที่ 3: นโยบายความปลอดภัย (Security Policy)

### 3.1 การเข้ารหัสข้อมูล

| ประเภทข้อมูล | วิธีการป้องกัน |
|-------------|--------------|
| เบอร์โทรศัพท์, เลขบัตรประชาชน | เข้ารหัส Fernet (AES-128-CBC) ระดับ field ก่อนเก็บลงฐานข้อมูล |
| หมายเหตุรายจ่าย (expenses.note) | เข้ารหัส Fernet ระดับ field ก่อนเก็บลงฐานข้อมูล |
| Gmail OAuth Token | เข้ารหัส Fernet ก่อนเขียนลงดิสก์ |
| OAuth Client Secret ของแอป (credentials.json) | เข้ารหัส Fernet บนดิสก์ (managed encrypted-at-rest storage) |
| API Keys ของผู้ใช้ | เข้ารหัส Fernet ก่อนเก็บลงฐานข้อมูล |
| Token files บนดิสก์ | สิทธิ์ไฟล์ 600 (เฉพาะ owner เท่านั้น) |
| ข้อความแชท | เก็บเป็น plaintext ใน SQLite + ลบตาม retention |
| ตำแหน่ง GPS | เก็บเป็น plaintext ใน SQLite + ลบตาม TTL |

### 3.2 การลดข้อมูลขั้นต่ำ (Data Minimization)

- **อีเมล:** ระบบไม่เก็บเนื้อหาอีเมล หัวเรื่อง ที่อยู่ผู้ส่ง หรือ domain ผู้ส่ง เก็บเฉพาะ message ID (สำหรับป้องกันสรุปซ้ำ) และสถานะว่ามีหัวเรื่องหรือไม่ (boolean)
- **บันทึกการใช้เครื่องมือ:** ไม่เก็บเนื้อหาจริงของ input/output เก็บเฉพาะประเภท (text/numeric/label), fingerprint (SHA-256 แบบย่อ), และขนาด
- **ข้อความ error:** ไม่เก็บ error message ดั้งเดิม เก็บเฉพาะประเภท error (timeout/auth/network) และข้อความทั่วไปที่ไม่มีข้อมูลอ่อนไหว
- **ข้อความ Telegram ที่มีข้อมูลอ่อนไหว:** เมื่อผู้ใช้ส่งเลขบัตรประชาชนหรือเบอร์โทรผ่าน /setid หรือ /setphone ระบบพยายามลบข้อความนั้นจาก Telegram โดยอัตโนมัติ

### 3.3 การลบข้อมูลถาวร (Hard Purge)

คำสั่ง `/delete_my_data confirm` จะลบข้อมูลจากตารางปฏิบัติการที่ผูกกับผู้ใช้ ได้แก่: users, chat_history, conversations, processed_emails, tool_logs, reminders, todos, expenses, user_locations, oauth_states, user_consents, user_api_keys, pending_messages, schedules, job_runs รวมถึงลบ Gmail token file บนดิสก์

อย่างไรก็ตาม ระบบจะเก็บข้อมูลขั้นต่ำใน `security_audit_logs` ต่อไว้โดยตั้งใจ เพื่อ governance, accountability และการสืบสวนเหตุการณ์

การลบนี้เป็นการลบถาวร ไม่สามารถกู้คืนได้

### 3.4 ข้อจำกัดที่ทราบ

1. **ข้อความแชทและตำแหน่ง GPS** ยังเก็บเป็น plaintext ในฐานข้อมูล SQLite (มีแผนประเมิน database-level encryption ในอนาคต)
2. **Telegram Chat ID** ใช้เป็น lookup หลักของระบบ ยังไม่สามารถเข้ารหัสได้
3. **ข้อมูลที่ส่งไปยังผู้ให้บริการ AI** อยู่ภายใต้นโยบายความเป็นส่วนตัวของผู้ให้บริการนั้น (Anthropic, Google)
4. **ข้อมูลที่ถูก backup** ก่อนการลบอาจยังคงอยู่ใน backup files จนกว่า backup จะถูก rotate
5. ระบบทำงานบนเครื่องของผู้ให้บริการ ความปลอดภัยทางกายภาพขึ้นกับสภาพแวดล้อมที่ deploy
