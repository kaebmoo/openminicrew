วิเคราะห์รูปนี้ซึ่งเป็นใบเสร็จ, บิล, หรือ slip การโอนเงิน
ดึงรายการสินค้า/บริการออกมาเป็น JSON:
{{"store": "<ชื่อร้าน>", "receipt_date": "<วันที่ในใบเสร็จเป็น YYYY-MM-DD ค.ศ. หรือ null ถ้าไม่ระบุ>", "subtotal": <ยอดรวมก่อน SC/VAT (number หรือ null)>, "grand_total": <ยอดสุทธิที่จ่ายจริง (number หรือ null)>, "items": [
  {{"amount": <ยอดเงินรวมของบรรทัดนั้น (number)>, "qty": <จำนวน (number, default 1)>, "category": "<หมวดหมู่>", "note": "<ชื่อรายการสั้นๆ>"}},
  ...
]}}

## โครงสร้างใบเสร็จไทย
ใบเสร็จมี 2 column: ชื่อรายการ (ซ้าย) กับ ราคา (ขวา)
- **รายการสินค้า**: มีชื่อสินค้าและราคาอยู่ใน column ขวา → สร้าง item
- **บรรทัดราคาต่อชิ้น**: เยื้องซ้าย แสดง 'จำนวน * ราคา / ชิ้น' เช่น '2 * 29.00 / ชิ้น' → เป็นรายละเอียดของรายการก่อนหน้า **ห้ามสร้าง item แยก** ราคารวมอยู่ในบรรทัดก่อนหน้าแล้ว
- **ส่วนลด/coupon**: เยื้องซ้าย มียอดติดลบ เช่น 'CPN3 - BHT  -31.75' → สร้าง item โดย amount เป็นจำนวนติดลบ

## กฎ
amount = ยอดเงินรวมของบรรทัดนั้นตามที่พิมพ์ใน column ราคา
ถ้า qty=2 ราคาชิ้นละ 30 แสดง 60 → ใส่ amount=60, qty=2
subtotal = ยอดรวมก่อนบวก service charge และ VAT (ถ้าไม่มีใส่ null)
grand_total = ยอดเงินสุทธิที่จ่ายจริง (ถ้าไม่มีใส่ null)
**receipt_date**: ดูวันที่ในใบเสร็จ/slip แล้วแปลงเป็น `YYYY-MM-DD` (ปี ค.ศ.)
- ใบเสร็จไทยมักใช้พ.ศ. — ต้องลบ 543 ออก เช่น "18/05/2569" → "2026-05-18"
- เดือนภาษาไทย "18 พฤษภาคม 2569" → "2026-05-18"
- ถ้าไม่ระบุวันที่ในใบเสร็จ ใส่ null (ห้ามเดาด้วยวันปัจจุบัน)
**ตรวจสอบ**: ผลรวมของ amount ทุกรายการ (รวมส่วนลดติดลบ) ต้องเท่ากับ grand_total
หมวดหมู่ที่แนะนำ: อาหาร, เครื่องดื่ม, เดินทาง, ช็อปปิ้ง, ของใช้, สาธารณูปโภค, สุขภาพ, บันเทิง, การศึกษา, โอนเงิน, ทั่วไป
ถ้าเป็น slip โอนเงินที่มีรายการเดียว ให้คืน items เพียง 1 ตัว
ตอบเฉพาะ JSON เท่านั้น ไม่ต้องอธิบายเพิ่ม
ถ้าอ่านไม่ออกหรือไม่ใช่ใบเสร็จ/บิล/slip ให้ตอบ: null
