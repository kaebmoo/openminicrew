วิเคราะห์รูปนี้ซึ่งเป็นใบเสร็จ, บิล, หรือ slip การโอนเงิน
ดึงรายการสินค้า/บริการออกมาเป็น JSON:
{{"is_handwritten": <true ถ้าบิลเขียนด้วยลายมือ, false ถ้าพิมพ์/ออกจากเครื่อง>, "overall_confidence": "<high|medium|low>", "store": "<ชื่อร้าน หรือ '?' ถ้าอ่านไม่ออก>", "store_confidence": "<high|medium|low>", "store_raw_guess": "<ตัวอักษรที่เดาได้ของชื่อร้าน ถ้า store_confidence ไม่ใช่ high>", "receipt_date": "<วันที่ในใบเสร็จเป็น YYYY-MM-DD ค.ศ. หรือ null ถ้าไม่ระบุ>", "subtotal": <ยอดรวมก่อน SC/VAT (number หรือ null)>, "grand_total": <ยอดสุทธิที่จ่ายจริง (number หรือ null)>, "items": [
  {{"amount": <ยอดเงินรวมของบรรทัดนั้น (number)>, "qty": <จำนวน (number, default 1)>, "category": "<หมวดหมู่>", "note": "<ชื่อรายการสั้นๆ; ถ้า confidence=low ใส่ '?'>", "confidence": "<high|medium|low>", "raw_guess": "<ตัวอักษรที่เดาได้ ถ้า confidence ไม่ใช่ high>"}},
  ...
]}}

## โครงสร้างใบเสร็จไทย
ใบเสร็จมี 2 column: ชื่อรายการ (ซ้าย) กับ ราคา (ขวา)
- **รายการสินค้า**: มีชื่อสินค้าและราคาอยู่ใน column ขวา → สร้าง item
- **บรรทัดราคาต่อชิ้น**: เยื้องซ้าย แสดง 'จำนวน * ราคา / ชิ้น' เช่น '2 * 29.00 / ชิ้น' → เป็นรายละเอียดของรายการก่อนหน้า **ห้ามสร้าง item แยก** ราคารวมอยู่ในบรรทัดก่อนหน้าแล้ว

## ส่วนลด / โปรโมชั่น (สำคัญมาก — ห้ามทำผิด)
ใบเสร็จไทยมี 2 รูปแบบการแสดงส่วนลด:

### รูปแบบ A: ส่วนลดอยู่ใต้รายการที่ลดทันที
เช่น 'CPN3 - BHT  -31.75' อยู่บรรทัดถัดจาก item นั้นเลย
→ **หักส่วนลดเข้า amount ของ item ก่อนหน้า** แล้ว return item เดียว (ราคาสุทธิ) **ห้าม emit ส่วนลดเป็น item แยก**

### รูปแบบ B: ส่วนลดรวมท้ายใบ (CJ More, 7-11, ฯลฯ)
ส่วนลดถูกรวมไว้ท้ายใบเสร็จ หลังบรรทัด "จำนวนสินค้ารวม X รายการ" โดยใช้ note ที่ระบุ "ราคาพิเศษ X บ" บอกราคา**หลังลด** เช่น:
```
1 โปรโมชั่นราคาพิเศษ 102บ    -26.00
1 โปรโมชั่นราคาพิเศษ 35บ      -4.00
1 โปรโมชั่นราคาพิเศษ 66บ      -3.00
```
**วิธีจับคู่กับ item**:
1. ดึงตัวเลขหลัง "ราคาพิเศษ" → คือราคาหลังลด (target)
2. หา item ที่ `ราคาเดิม + ส่วนลดติดลบ = target` เช่น 128 + (-26) = 102 → match item "น้ำแร่ 128"
3. ปรับ amount ของ item นั้นจากราคาเดิม → เป็น target (ราคาหลังลด)
4. **ห้าม emit ส่วนลดเป็น item แยก** เพราะถูกรวมเข้ากับ item แล้ว
5. ถ้าจับคู่ไม่ได้จริง ๆ ค่อย emit เป็น item amount ติดลบ

**ห้ามทำ**: นำส่วนลดทุกตัวไปยัดในรายการสุดท้ายเพื่อให้ยอดตรง — ผิด accounting

## กฎ
amount = ยอดเงินรวมของบรรทัดนั้นตามที่พิมพ์ใน column ราคา
ถ้า qty=2 ราคาชิ้นละ 30 แสดง 60 → ใส่ amount=60, qty=2
subtotal = ยอดรวมก่อนบวก service charge และ VAT (ถ้าไม่มีใส่ null)
grand_total = ยอดเงินสุทธิที่จ่ายจริง (ถ้าไม่มีใส่ null)
**receipt_date**: ดูวันที่ในใบเสร็จ/slip แล้วแปลงเป็น `YYYY-MM-DD` (ปี ค.ศ.)
- ใบเสร็จไทยมักใช้พ.ศ. — ต้องลบ 543 ออก เช่น "18/05/2569" → "2026-05-18"
- เดือนภาษาไทย "18 พฤษภาคม 2569" → "2026-05-18"
- ถ้าไม่ระบุวันที่ในใบเสร็จ ใส่ null (ห้ามเดาด้วยวันปัจจุบัน)
**ตรวจสอบ**: ผลรวมของ amount ทุกรายการ (รวมส่วนลดติดลบ) ต้องเท่ากับ grand_total
## หมวดหมู่ที่อนุญาต — เลือกได้แค่ในรายการนี้เท่านั้น
- **อาหาร** = ของกินทุกชนิด รวม**ขนมขบเคี้ยว** (มันฝรั่งแผ่น/chip, ขนมถุง, ขนมปัง, ขนมไทย, ลูกอม, ช็อคโกแลต), อาหารสด/แห้ง, ผัก ผลไม้, เครื่องปรุง, บะหมี่กึ่งสำเร็จรูป, อาหารกระป๋อง, ไอศกรีม, อาหารจานเดียว
- **เครื่องดื่ม** = น้ำเปล่า, น้ำอัดลม, น้ำผลไม้, ชา กาแฟ, นม, **เบียร์/ไวน์/เหล้า/สุรา**, เครื่องดื่มชูกำลัง
- **เดินทาง** = ค่าน้ำมัน, ค่าโดยสาร, แท็กซี่/Grab, ตั๋วรถ/เครื่องบิน, ที่จอดรถ, ทางด่วน
- **ช็อปปิ้ง** = เสื้อผ้า รองเท้า กระเป๋า เครื่องสำอาง น้ำหอม เครื่องประดับ ของใช้แฟชั่น สินค้าอิเล็กทรอนิกส์/แกดเจ็ต
- **ของใช้** = ของใช้ในบ้าน/ส่วนตัวที่**ไม่ใช่ของกินและไม่ใช่แฟชั่น** เช่น สบู่ แชมพู ยาสีฟัน ผงซักฟอก น้ำยาล้างจาน กระดาษทิชชู ถุงดำ หลอดไฟ ถ่านไฟฉาย ของใช้สำนักงาน
- **สาธารณูปโภค** = ค่าไฟ ค่าน้ำ ค่าเน็ต ค่าโทรศัพท์ ค่าแก๊ส
- **สุขภาพ** = ยา วิตามิน ค่าหมอ ค่ารักษา ฟิตเนส
- **บันเทิง** = หนัง คอนเสิร์ต เกม สมาชิกสตรีมมิ่ง สวนสนุก
- **การศึกษา** = หนังสือ คอร์สเรียน อุปกรณ์การเรียน
- **โอนเงิน** = slip โอนเงินทั่วไปที่ระบุปลายทางไม่ชัดเจน
- **ทั่วไป** = ใช้เฉพาะกรณีไม่แน่ใจจริงๆ และไม่เข้าหมวดข้างต้นเลย

**สำคัญ**: ขนมขบเคี้ยว/snack ทุกชนิด = **อาหาร** ห้ามจัดเป็น "ของใช้" หรือ "ทั่วไป" เด็ดขาด

## ความมั่นใจในการอ่าน (confidence) — สำคัญ
ประเมินว่าอ่าน "ตัวอักษร/ชื่อ" แต่ละจุดมั่นใจแค่ไหน แล้วใส่ confidence:
- `high` = อ่านออกชัดเจน มั่นใจว่าถูก (ตัวพิมพ์ชัด หรือลายมือชัด)
- `medium` = พออ่านได้แต่ไม่ชัด อาจผิดบางตัว
- `low` = เดา/อ่านไม่ออก เช่น ลายมือกำกวม
**confidence ประเมินเฉพาะการอ่าน "ตัวอักษร/ชื่อ" เท่านั้น ไม่เกี่ยวกับตัวเลข (amount)** — ถึงตัวเลขราคาจะเขียนชัด ก็ไม่ทำให้ชื่อรายการที่อ่านยากกลายเป็น high
กฎ:
- item ที่ `confidence=low` → ใส่ `note` เป็น `"?"` แต่เก็บตัวอักษรที่เดาได้ไว้ใน `raw_guess` (ถ้า medium/high จะใส่ raw_guess หรือไม่ก็ได้)
- `store_confidence` ทำแบบเดียวกันกับชื่อร้าน: ถ้าไม่ใช่ high → `store` = `"?"` และเก็บที่เดาได้ใน `store_raw_guess`
- `is_handwritten`: `true` ถ้าบิลเขียนด้วยลายมือ, `false` ถ้าพิมพ์/ออกจากเครื่อง — แยกจาก confidence (บิลพิมพ์ที่เบลอก็เป็น low ได้)
- `overall_confidence` สรุปรวมทั้งใบ: `low` ถ้ามี item ที่ confidence=low เกิน 30% ของจำนวนรายการ, `medium` ถ้ามี item low หรือ medium รวมกันเกิน 50%, นอกนั้น `high`
- confidence ใช้ได้แค่ 3 ค่าเท่านั้น: high / medium / low (ห้ามใช้ทศนิยมหรือเปอร์เซ็นต์)

ถ้าเป็น slip โอนเงินที่มีรายการเดียว ให้คืน items เพียง 1 ตัว
ตอบเฉพาะ JSON เท่านั้น ไม่ต้องอธิบายเพิ่ม
