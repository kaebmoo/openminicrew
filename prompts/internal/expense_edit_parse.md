ผู้ใช้กำลังแก้ไขรายการในบิลที่ระบบอ่านจากรูป (OCR) มาก่อนหน้านี้
รายการปัจจุบัน (บรรทัดเริ่มนับที่ 1):
{bill_state}

ข้อความของผู้ใช้:
"{user_message}"

หน้าที่ของคุณ: แปลงคำสั่งของผู้ใช้เป็น operations เพื่อแก้รายการ ตอบเป็น JSON เท่านั้น:
{{"operations": [
  {{"op": "edit", "line": <เลขบรรทัด 1..N>, "note": "<ชื่อใหม่ ถ้าระบุ>", "amount": <ราคาใหม่ number ถ้าระบุ>, "category": "<หมวด ถ้าระบุ>"}},
  {{"op": "delete", "line": <เลขบรรทัด 1..N>}},
  {{"op": "add", "note": "<ชื่อรายการ>", "amount": <ราคา number>, "category": "<หมวด>"}}
], "confidence": "<high|low>"}}

กฎ:
- `op` มี 3 แบบ: `edit` (แก้บรรทัดที่มีอยู่), `delete` (ลบบรรทัด), `add` (เพิ่มรายการใหม่)
- `line` ต้องเป็นเลขบรรทัดจริงในรายการปัจจุบัน (1 ถึง N) สำหรับ `edit` และ `delete` เท่านั้น — `add` ไม่ต้องมี line
- ใส่เฉพาะ field ที่ผู้ใช้ระบุจริง เช่น แก้แค่ชื่อ → ใส่แค่ `note` ไม่ต้องใส่ `amount`/`category`
- `amount` เป็นตัวเลขบวกเสมอ (ถ้าระบุ)
- `category` ถ้าระบุ ต้องเลือกจากนี้เท่านั้น: อาหาร, เครื่องดื่ม, เดินทาง, ช็อปปิ้ง, ของใช้, สาธารณูปโภค, สุขภาพ, บันเทิง, การศึกษา, โอนเงิน, ทั่วไป
- `confidence` = `high` ถ้ามั่นใจว่าเข้าใจคำสั่งถูกต้อง, `low` ถ้าไม่แน่ใจหรือคำสั่งกำกวม
- ถ้าตีความคำสั่งไม่ได้เลย → ตอบ {{"operations": [], "confidence": "low"}}
- ตอบเฉพาะ JSON เท่านั้น ห้ามอธิบายเพิ่ม
